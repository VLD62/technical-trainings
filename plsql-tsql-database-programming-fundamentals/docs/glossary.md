# Glossary

**Anonymous block** — A PL/SQL block executed without being stored as a named database object.

**Batch** — A group of T-SQL statements submitted together by a client. `GO` commonly separates batches in client tools.

**Bind parameter** — A placeholder whose value is supplied separately from SQL text, improving safety and plan reuse.

**Cardinality** — The number of rows expected or produced by an operation.

**Constraint** — A database rule such as primary key, foreign key, unique, check, or not-null.

**Cursor** — A mechanism for iterating through a query result row by row.

**DDL** — Data Definition Language used to create or change database objects.

**DML** — Data Manipulation Language used to insert, update, delete, or merge data.

**Exception** — PL/SQL mechanism for representing and handling runtime errors.

**Function** — A stored program unit that returns a value and may be usable inside SQL expressions, subject to platform rules.

**Package** — Oracle object that groups a public specification and implementation body for related PL/SQL elements.

**PL/SQL** — Oracle's procedural extension to SQL.

**Procedure** — A named stored program unit that performs an operation and can accept parameters.

**Savepoint** — A named point inside a transaction to which part of the work can be rolled back.

**Schema** — A namespace and ownership boundary for database objects. Oracle and SQL Server use the term differently.

**Set-based operation** — A SQL operation that processes a set of rows as a unit instead of looping row by row.

**Stored program unit** — A procedure, function, package, or trigger stored in the database.

**T-SQL** — Microsoft's Transact-SQL extension used by SQL Server and related services.

**Transaction** — A unit of work that succeeds or fails according to atomicity and consistency rules.

**Trigger** — Database code that executes automatically in response to a defined event.

**Three-valued logic** — SQL logic in which a predicate can be true, false, or unknown because of NULL.

**XACT_STATE()** — SQL Server function that reports whether the current transaction is absent, active and committable, or active but uncommittable.
