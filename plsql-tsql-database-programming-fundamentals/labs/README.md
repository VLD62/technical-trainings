# Labs

The labs use the same employee-management model on Oracle and SQL Server.

## Recommended Order

1. Start and verify databases
2. Query the shared schema
3. Work with blocks, variables, and control flow
4. Create and execute procedures, functions, and packages
5. Compare cursors with set-based processing
6. Handle exceptions and transactions
7. Audit changes and complete the final challenge

## Rules

- Run every task on both platforms unless marked otherwise.
- Predict the result before execution.
- Record syntax differences and behavioral differences separately.
- Use explicit transactions for destructive experiments.
- Reset the local environment with `make reset` when necessary.
- Never run these labs against production data.
