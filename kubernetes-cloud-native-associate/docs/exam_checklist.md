# KCNA Exam Readiness Checklist

## Kubernetes Fundamentals — 44%

- [ ] I can explain the roles of the API server, etcd, scheduler, controller manager, kubelet, runtime, and network dataplane.
- [ ] I can explain desired state and reconciliation.
- [ ] I can distinguish a Pod from a controller-managed workload.
- [ ] I can select Deployment, StatefulSet, DaemonSet, Job, or CronJob for a scenario.
- [ ] I can explain namespaces, labels, selectors, and annotations.
- [ ] I can explain requests, limits, affinity, taints, and tolerations.
- [ ] I can distinguish ConfigMaps, Secrets, environment variables, and mounted configuration.
- [ ] I can explain container images, registries, tags, digests, and runtimes.
- [ ] I can use API discovery and read a basic manifest.

## Container Orchestration — 28%

- [ ] I can explain Pod networking and Service discovery.
- [ ] I can distinguish ClusterIP, NodePort, LoadBalancer, ExternalName, and headless Services.
- [ ] I can explain the purpose of DNS, Ingress, Gateway API, and service mesh.
- [ ] I can explain authentication, authorization, admission, and RBAC.
- [ ] I understand that Base64 does not encrypt a Secret.
- [ ] I can explain the purpose and enforcement dependency of NetworkPolicy.
- [ ] I can distinguish Volume, PV, PVC, StorageClass, dynamic provisioning, and CSI.
- [ ] I can troubleshoot Pending, ImagePullBackOff, CrashLoopBackOff, selector mismatch, and authorization errors.

## Cloud Native Application Delivery — 16%

- [ ] I can distinguish CI, continuous delivery, and continuous deployment.
- [ ] I can explain the GitOps source-of-truth and reconciliation model.
- [ ] I can explain Helm charts, values, templates, and releases.
- [ ] I can compare rolling, recreate, blue/green, and canary strategies.
- [ ] I can distinguish startup, readiness, and liveness probes.
- [ ] I can explain rollout history, status, and rollback limitations.
- [ ] I can describe basic software supply-chain controls.

## Cloud Native Architecture — 12%

- [ ] I can distinguish metrics, logs, traces, events, and profiles.
- [ ] I can explain the purposes of Prometheus and OpenTelemetry.
- [ ] I can explain monitoring versus observability.
- [ ] I can recognize major CNCF ecosystem categories.
- [ ] I can explain Sandbox, Incubating, and Graduated project maturity.
- [ ] I can explain the purposes of OCI, CRI, CNI, and CSI.
- [ ] I can describe open governance and common contribution paths.

## Practical Confidence

- [ ] I can inspect cluster resources with `kubectl get`.
- [ ] I can use `describe`, events, and logs to investigate a failure.
- [ ] I can identify a Service selector mismatch.
- [ ] I can explain why a PVC might remain Pending.
- [ ] I can safely create and remove a training namespace.

## Exam Logistics

- [ ] I reviewed the current official certification page.
- [ ] I reviewed the current candidate handbook and important instructions.
- [ ] I tested the required secure browser and system compatibility.
- [ ] I confirmed acceptable identification documents.
- [ ] I prepared a compliant room, desk, camera, microphone, and network connection.
- [ ] I verified the scheduled date, time zone, and rescheduling deadline.
- [ ] I understand the current retake and certification-validity policies.
