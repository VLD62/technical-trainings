# Exam 101 Review

### Consolidation for topics 101–104

**Presenter:** Vladislav Iliev

**Audience:** Linux learners, system administrators, DevOps engineers, and certification candidates

---

## High-Weight Areas

- Command-line work, file management, streams, and processes carry significant weight.
- Package management and mounting require command recognition and practical understanding.
- Do not neglect boot targets, regular expressions, vi, and permissions.

---

## Recognition vs Recall

- Recognize command options from realistic output.
- Recall important files and directories.
- Explain command side effects.
- Distinguish similar tools and package workflows.

---

## Troubleshooting Scenarios

- System does not reach the expected target.
- Package owns an unknown file.
- Filesystem is full despite small visible files.
- A mount fails after reboot.
- A process ignores normal termination.

---

## Practical Review

- Build and inspect a loopback filesystem.
- Archive and checksum a directory.
- Filter log data using a pipeline.
- Create and inspect hard and symbolic links.
- Change process priority safely.

---

## Exam Technique

- Read every option before selecting.
- Pay attention to absolute paths, redirections, and argument order.
- Eliminate commands that are valid but do not satisfy the exact task.
- Do not overthink commands you have practiced repeatedly.

---

## Readiness Check

- Can you complete the 101 checklist without notes?
- Can you explain commands, not only copy them?
- Can you recover from a deliberate lab mistake?
- Can you complete a timed mixed quiz?

---

## Practice

- Run the commands in a disposable lab.
- Capture expected and unexpected output.
- Explain what changed and how to reverse it.
- Complete the matching lab and checklist items.

---

## Key Takeaways

- Understand the concept before memorizing options.
- Know the important commands, files, and failure modes.
- Prefer safe, observable, reversible practice.
- Review the official LPIC-1 objectives as the source of truth.
