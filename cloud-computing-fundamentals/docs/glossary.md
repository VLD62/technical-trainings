# Cloud Computing Glossary

**Account / Subscription / Project / Compartment** — Provider-specific administrative and isolation boundary used for resources, permissions, quota, or billing.

**API** — Programmatic interface used to manage or consume cloud services.

**Availability Zone / Availability Domain** — Isolated infrastructure location inside a region.

**Backup** — Recoverable copy of data retained separately from the primary working copy.

**CaaS** — Containers as a Service; managed container platform or orchestration capability.

**Cloud bursting** — Temporarily using external cloud capacity when another environment reaches limits.

**Cloud-native** — Design and operating approach that uses automation, elasticity, managed services, failure-aware architecture, and continuous delivery.

**Community cloud** — Infrastructure shared by organizations with common requirements.

**Consumption pricing** — Payment based on measured usage.

**Control plane** — APIs and services used to configure and manage resources.

**Data plane** — Components that process application traffic or data.

**Data residency** — Requirement or decision concerning the geographic location of data.

**DBaaS** — Database as a Service.

**Disaster recovery** — Processes and architecture used to restore services after major disruption.

**Edge computing** — Processing close to users, devices, or data sources.

**Elasticity** — Dynamic increase or decrease of capacity according to demand.

**Fault domain** — Infrastructure grouping designed to isolate a smaller class of failure.

**FaaS** — Function as a Service; event-driven execution with provider-managed infrastructure.

**FinOps** — Operational practice that creates shared accountability for cloud value and cost.

**High availability** — Design that reduces interruption from expected component failures.

**Hybrid cloud** — Integrated use of public cloud and private or on-premises environments.

**IaaS** — Infrastructure as a Service.

**Identity federation** — Trust relationship that allows an external identity provider to authenticate users or workloads.

**Immutable infrastructure** — Replacing infrastructure with a new version rather than modifying it in place.

**Infrastructure as Code** — Version-controlled definitions used to provision and configure infrastructure.

**Landing zone** — Reusable foundation for identity, networking, governance, logging, security, and billing.

**Least privilege** — Granting only the minimum required access for the minimum required time.

**Managed identity / workload identity** — Provider-managed identity used by an application or resource without embedded long-lived credentials.

**Multi-cloud** — Deliberate use of services from more than one cloud provider.

**Multi-tenancy** — Sharing underlying service infrastructure between isolated customers or workloads.

**PaaS** — Platform as a Service.

**Policy as Code** — Automated evaluation or enforcement of configuration rules.

**Private cloud** — Cloud operating model dedicated to one organization.

**Public cloud** — Provider-operated cloud services offered to many customers.

**Region** — Geographic area where a provider operates cloud infrastructure.

**RPO** — Recovery Point Objective; maximum acceptable data loss measured in time.

**RTO** — Recovery Time Objective; maximum acceptable service restoration time.

**SaaS** — Software as a Service.

**Scalability** — Ability to handle increased demand by adding capacity.

**Serverless** — Service model where underlying capacity and much platform operation are provider-managed; customer operational responsibility still exists.

**Shared responsibility model** — Division of security and operational responsibilities between provider and customer.

**Sovereign cloud** — Environment designed to meet specific jurisdictional, residency, operator-control, or sovereignty requirements.

**Tag / Label** — Resource metadata used for ownership, governance, automation, and cost allocation.

**Vendor lock-in** — Cost or difficulty of moving from a provider-specific technology or commercial relationship.

**Virtual network** — Software-defined isolated network in a cloud platform.

**Workload** — Application, service, data platform, or technical capability operated in an environment.
