# Labs

The labs use the fictional ShopNow workload from `examples/workload-requirements.yaml`.

| Lab | Topic | Account required |
|---|---|---|
| 01 | Classify services and deployment models | No |
| 02 | Design a highly available web application | No |
| 03 | Design identity and least privilege | No |
| 04 | Map architecture to AWS, Azure, Google Cloud, and OCI | No |
| 05 | Review cost allocation and tagging | No |
| 06 | Make a multi-cloud architecture decision | No |

For each lab, record:

- Assumptions
- Selected architecture
- Shared-responsibility boundary
- Main risks
- Cost drivers
- Validation or testing approach

There is rarely one perfect answer. The goal is to make trade-offs explicit and defensible.
