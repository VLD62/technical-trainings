# Quiz 01 — Cloud Concepts

## Questions

1. What is the difference between scalability and elasticity?
2. Which cloud characteristic makes API-driven provisioning possible without a manual infrastructure ticket?
3. In IaaS, who normally manages the guest operating system?
4. Why is serverless not the same as having no operational responsibility?
5. What is the difference between public cloud and private cloud?
6. What is hybrid cloud?
7. Why can rehosting a VM fail to capture major cloud benefits?
8. Give one valid and one weak reason for multi-cloud.
9. Which service model normally gives the customer the most infrastructure control?
10. What is measured consumption?

## Answers

1. Scalability is the ability to add capacity; elasticity is dynamic adjustment of capacity according to demand.
2. On-demand self-service through provider APIs and automation.
3. The customer.
4. The customer still owns code, configuration, identity, data, observability, limits, and failure behavior.
5. Public cloud is provider-operated and shared across customers; private cloud is dedicated to one organization while using cloud operating characteristics.
6. Integrated use of public cloud and private or on-premises environments.
7. It can preserve manual operations, fixed capacity, monolithic design, and high operational responsibility.
8. Valid: regulatory requirement or unique provider capability. Weak: vague fear of lock-in without workload analysis.
9. IaaS.
10. Usage is metered and can be reported or billed according to consumed resources.
