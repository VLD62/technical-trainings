# AI Agents & Workflows Glossary

## Agent

Software system that uses an LLM, tools, memory, and logic to perform tasks and achieve goals.

## LLM (Large Language Model)

AI model trained on large amounts of text data capable of understanding and generating natural language.

## Prompt

Instructions provided to an LLM that define its behavior and task.

## Tool

External capability available to an agent, such as an API, function, script, or database query.

## Tool Calling

The ability of an LLM to decide when and how to use a tool.

## Memory

Mechanism for storing and retrieving information across interactions.

## Context Window

The amount of information an LLM can process in a single request.

## Workflow

A predefined sequence of steps executed to achieve a task.

## LangChain

Framework for building applications powered by LLMs.

## LangGraph

Framework for creating agent workflows using graph-based execution.

## LangSmith

Platform for tracing, debugging, and evaluating LLM applications.

## RAG (Retrieval Augmented Generation)

Technique that combines LLMs with external knowledge sources.

## Vector Database

Database optimized for storing and searching vector embeddings.

## Embedding

Numerical representation of text used for semantic search.

## Human-in-the-Loop (HITL)

Approach where human feedback or approval is included in the execution process.

## Multi-Agent System

System consisting of multiple agents working together to solve tasks.

## Observability

The ability to inspect and understand how an AI application behaves internally.

## Tracing

Recording execution steps of an AI application for debugging and analysis.
