# Glossary

## Artifact

A file produced, consumed, or distributed during software delivery. Examples include packages, binaries, archives, libraries, reports, and release bundles.

## Binary repository manager

A system used to store, organize, control, and distribute software artifacts.

## Local repository

A repository that stores artifacts produced by your organization or project.

## Remote repository

A repository that proxies and caches artifacts from an external upstream source.

## Virtual repository

A repository that provides one access point across multiple local and remote repositories.

## Promotion

The process of moving or marking an artifact as approved for a later lifecycle stage, such as release or production.

## Checksum

A calculated value used to verify that a file was not changed or corrupted.

## Retention policy

Rules defining how long artifacts are kept before cleanup or archival.

## Immutable artifact

An artifact that is not overwritten after it has been published.

## Access group

A group used to manage repository permissions for multiple users.
