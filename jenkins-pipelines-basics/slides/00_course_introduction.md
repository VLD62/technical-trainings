# Jenkins Pipelines: Basics

### Course Introduction

**Audience:** Developers, DevOps Engineers, QA Engineers, Release Engineers, IT Professionals

---

# About This Course

This course provides a practical introduction to Jenkins and pipeline-based CI/CD automation.

The goal is to understand:

* What Jenkins is
* How Jenkins fits into CI/CD
* How Jenkins controllers, agents, jobs, plugins, and pipelines work together
* How to write maintainable Jenkinsfiles
* Which pipeline practices help teams avoid common problems

---

# Why This Course?

Jenkins is widely used to automate software delivery workflows.

It can help teams automate:

* Build execution
* Test execution
* Static checks
* Artifact publishing
* Deployment workflows
* Notifications and feedback loops

---

# Learning Objectives

By the end of this course you will be able to:

✅ Explain the basic Jenkins architecture  
✅ Understand Jenkins plugins and integrations  
✅ Write a basic Declarative Pipeline  
✅ Understand when Scripted Pipeline may be needed  
✅ Use stages, steps, parameters, and environment variables  
✅ Apply Jenkins pipeline best practices

# Prerequisites

Recommended knowledge:

* Basic Git usage
* Basic Linux command line knowledge
* Basic software build/test concepts
* Basic YAML/Groovy-like syntax familiarity is helpful, but not required

---

# Course Philosophy

This is a practical course.

We will focus on:

* Understanding the concepts
* Reading Jenkinsfiles
* Writing small examples
* Discussing maintainability
* Avoiding hardcoded and fragile pipelines
