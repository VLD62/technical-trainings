# Jenkins Glossary

## Agent

A machine, container, or runtime that executes Jenkins pipeline work.

## Build

One execution of a Jenkins job or pipeline.

## Controller

The main Jenkins node that manages configuration, scheduling, plugins, jobs, credentials, and agents.

## Declarative Pipeline

A structured Jenkins Pipeline syntax designed to make common CI/CD pipelines easier to read and maintain.

## Jenkinsfile

A text file that defines a Jenkins Pipeline as code.

## Job

A Jenkins configuration that defines work to execute.

## Node

A Jenkins execution environment. In Scripted Pipeline, `node` allocates an executor and workspace.

## Parameter

A user-provided or predefined value that changes how a pipeline runs.

## Pipeline

A Jenkins automation workflow defined as code.

## Plugin

An extension that adds functionality to Jenkins.

## Post Action

A pipeline section used to run actions after pipeline or stage completion, such as cleanup, report publishing, or notifications.

## Scripted Pipeline

A more flexible Jenkins Pipeline syntax that uses Groovy more directly.

## Stage

A logical section of a pipeline, such as Build, Test, Package, or Deploy.

## Step

An individual action inside a pipeline stage.

## Workspace

The directory where Jenkins checks out source code and runs commands for a build.
