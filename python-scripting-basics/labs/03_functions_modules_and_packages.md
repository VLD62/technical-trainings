# Lab 3: Functions, Modules, and Packages

## Tasks

1. Split calculations, formatting, and CLI behavior into modules.
2. Add type hints and docstrings.
3. Reject missing required record fields.
4. Avoid mutable global state.
5. Run the tool through `python -m package.module`.

## Stretch Task

Add a reusable package-level API.

## Review

- Which responsibility belongs in a function or class?
- Which errors are expected user errors?
- How will the behavior be tested?
