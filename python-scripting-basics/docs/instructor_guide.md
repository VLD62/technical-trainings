# Instructor Guide

## Delivery Options

- Ten 90-minute sessions
- Five half-day workshops
- Two intensive training days plus self-study
- Four-week internal learning program

## Suggested Session Flow

1. Introduce objectives
2. Explain the core concept
3. Run a live example
4. Add a controlled defect
5. Let participants diagnose it
6. Complete the lab
7. Review trade-offs
8. Finish with knowledge-check questions

## Teaching Priorities

Emphasize readability, explicit error handling, small functions, portable paths, composition before inheritance, testable structure, and safe automation.

## Common Misconceptions

- Dynamic typing means that values have no type
- Every script should be converted into classes
- Inheritance is the main purpose of OOP
- Printing an error is the same as handling it
- Small scripts do not need tests
- Type hints enforce runtime types
