# Glossary

**Argument:** A value passed to a function or command-line program.

**Attribute:** A value stored on an object or class.

**Class:** A definition of related object state and behavior.

**Composition:** Building behavior by combining collaborating objects.

**Dataclass:** A class decorated with `@dataclass` to generate common methods.

**Dependency injection:** Providing dependencies from outside an object.

**Dictionary:** A mutable key-value collection.

**Exception:** An object representing an error or unusual condition.

**Function:** A reusable block of code that can return a value.

**Idempotent:** Producing the same desired state when executed repeatedly.

**Inheritance:** Defining a class based on another class.

**Instance:** A concrete object created from a class.

**Module:** An importable Python source file.

**Package:** A directory grouping Python modules.

**Protocol:** A behavior-based type interface.

**Set:** A collection of unique values.

**Tuple:** An ordered immutable collection.

**Type hint:** Metadata describing an expected value type.
