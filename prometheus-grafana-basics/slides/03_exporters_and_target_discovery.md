# Session 3 – Exporters and Target Discovery

## Exporter Pattern

An exporter exposes metrics for a system that does not natively provide Prometheus-format metrics.

Examples:

- Node Exporter for Linux host metrics
- Blackbox Exporter for endpoint probing
- Database exporters
- Hardware and network exporters

---

## Node Exporter

Node Exporter exposes hardware and operating-system metrics.

Common metric families:

```text
node_cpu_seconds_total
node_memory_MemAvailable_bytes
node_filesystem_avail_bytes
node_network_receive_bytes_total
node_load1
node_uname_info
```

Default endpoint:

```text
http://HOST:9100/metrics
```

---

## Static Configuration

```yaml
scrape_configs:
  - job_name: linux-hosts
    static_configs:
      - targets:
          - host-01.example.internal:9100
          - host-02.example.internal:9100
```

Static configuration is simple but becomes difficult to maintain at scale.

---

## File-Based Service Discovery

Prometheus can read target lists from separate files.

Prometheus configuration:

```yaml
scrape_configs:
  - job_name: linux-hosts
    file_sd_configs:
      - files:
          - /etc/prometheus/targets/*.yml
```

Target file:

```yaml
- targets:
    - host-01.example.internal:9100
    - host-02.example.internal:9100
  labels:
    environment: training
    role: build-machine
```

This separates target inventory from the main Prometheus configuration.

## Target Health

The `up` metric is generated by Prometheus:

```promql
up{job="linux-hosts"}
```

Values:

- `1` – scrape succeeded
- `0` – scrape failed

The Prometheus targets page shows:

- Endpoint
- Labels
- Last scrape
- Scrape duration
- Error message
- Health state

---

## Troubleshooting Workflow

1. Check network connectivity.
2. Open the raw `/metrics` endpoint.
3. Check the target in Prometheus.
4. Inspect the last scrape error.
5. Validate labels and job names.
6. Check firewall and access rules.
7. Review exporter logs.
8. Verify time synchronization.

---

## Security

Do not expose Node Exporter directly to untrusted networks.

Use:

- Network segmentation
- Host firewall rules
- Reverse proxy or secure tunnel where appropriate
- Authentication and TLS for exposed monitoring endpoints
- Least-privilege access
